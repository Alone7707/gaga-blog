# UI 设计规范（清爽版）

## 1. 文档目的

本规范用于为当前开源博客产品重新定义一套新的视觉方向。

这一次不是在上一轮深色、玻璃感、控制台气质上继续微调，而是**明确推翻上一版偏厚重、偏深色、偏后台模板化的审美路线**，重建一套更适合内容产品长期使用的 UI 语言：

**年轻、简约、清爽、轻盈、现代内容产品感。**

目标不是做成甜腻的浅色运营页，也不是做成空洞的极简白板，而是建立一套：

- 看起来更轻、更透气
- 信息层级更清楚
- 更适合阅读与浏览
- 对前台内容和后台运营都成立
- 明显摆脱旧式后台、深色 SaaS 模板感

---

## 2. 新版设计总方向

### 2.1 核心设计概念

**设计关键词：年轻内容产品 / 轻编辑感 / 清爽工作台 / 有呼吸感的现代界面**

这套产品不再强调“深夜控制台”“冷峻技术编辑部”，而是转向：

- 前台像一个更现代的内容站点首页与阅读产品
- 后台像一个轻量、高效、清晰的运营工作台
- 前后台共享统一的年轻化基础设计语言
- 用轻盈感、留白和节奏取代厚重面板堆叠

### 2.2 风格结论

- **前台**：明亮、简洁、轻编辑感，强调内容可读性与浏览流动性
- **后台**：轻面板、低压迫感、高扫描效率，强调工作信息分层
- **共通语言**：浅色底、低饱和品牌色、柔和边框、大留白、明确字体层级、轻卡片、克制动效

### 2.3 一句话定义

> 这是一套更像现代内容产品、而不是传统后台模板的 UI 体系。

---

## 3. 与上一版的差异

### 3.1 上一版的问题判断

上一版的深色方案并不是不能用，但存在几个明显问题：

1. **整体过重**：页面大面积深底叠加重卡片，压迫感强
2. **后台感过强**：前台也容易看起来像管理系统外壳
3. **玻璃感偏多**：blur、渐变、发光边界叠加后，显得旧一代“科技后台风”
4. **信息层级被壳层吞掉**：容器存在感过强，内容本身不够突出
5. **阅读氛围不够轻松**：长文、列表、评论在深色环境里长期浏览疲劳感更高
6. **年轻感不足**：更像成熟、稳重、偏传统 SaaS，不像现代内容产品

### 3.2 新版明确保留什么

保留以下合理部分：

- 统一 token 的思路
- 前后台共享同一套基础语言的思路
- 卡片与状态体系的结构化思维
- 页面信息层级重构的方向

### 3.3 新版明确移除什么

以下旧风格应明确移除或大幅弱化：

- 大面积深色背景作为全站默认世界观
- 强玻璃拟态、重 blur、重阴影、重壳层边界
- 大量“青蓝发光 + 紫蓝渐变”的科技风装饰
- 首页和后台都使用厚重悬浮大卡片堆满屏幕
- 强烈“控制台”“深夜模式”“运维后台”语义
- 过于密集的描边、发光 hover、面板包裹层
- 看起来像默认管理系统的统计卡排列方式

### 3.4 新版替换关系

| 旧方向 | 新方向 |
|---|---|
| 深色控制台 | 明亮内容产品 |
| 厚重卡片 | 轻卡片 / 轻分组 |
| 发光科技风 | 低饱和品牌点缀 |
| 玻璃拟态 | 干净平面 + 柔和层次 |
| 面板主导 | 内容主导 |
| 强后台感 | 轻运营工作台感 |

## 4. 品牌与视觉气质

### 4.1 用户第一印象

用户进入产品后，应感受到：

- 这是一个现代、年轻的内容产品
- 内容和阅读体验被认真设计过
- 后台不是传统管理系统，而是轻量工作台
- 产品有节制、有审美，但不装饰过度

### 4.2 识别锚点

建议建立 4 个新版识别锚点：

1. **明亮轻底色体系**：整体干净，不脏不灰不沉闷
2. **有节奏的留白**：页面松弛，不是组件挤满容器
3. **温和品牌色点缀**：蓝绿系轻强调，不满屏抢戏
4. **内容优先排版**：标题、摘要、正文、辅助信息层次非常清楚

### 4.3 气质边界

新版气质应满足：

- 年轻，但不幼稚
- 简约，但不空
- 清爽，但不薄弱
- 现代，但不网红模板化
- 内容产品感强，但依旧有工具效率

---

## 5. 色彩系统

### 5.1 总原则

- 使用**浅色 / 近白底色**作为主世界观
- 强调色只用于关键交互与信息聚焦
- 避免过高饱和、高对比、大片渐变抢占内容注意力
- 前台与后台同源，但前台更柔和，后台更结构化

### 5.2 推荐主色板

#### 中性色

- `bg-page`: #F6F8FB —— 页面主背景
- `bg-soft`: #F2F5F9 —— 柔和区块背景
- `bg-card`: #FFFFFF —— 主卡片背景
- `bg-card-soft`: #FAFBFD —— 次卡片背景
- `line-soft`: #E7ECF3 —— 常规分隔线
- `line-strong`: #D9E2EE —— 重要边界

#### 文本色

- `text-1`: #172033 —— 主标题 / 核心文字
- `text-2`: #344054 —— 正文 / 主信息
- `text-3`: #667085 —— 次级信息
- `text-4`: #98A2B3 —— 弱提示 / 说明信息

#### 品牌强调色

- `accent-primary`: #4C8BF5 —— 主品牌蓝
- `accent-primary-soft`: #EAF2FF —— 主品牌浅底
- `accent-mint`: #54C6A9 —— 清爽薄荷绿，用于轻成功与内容活力
- `accent-sky`: #7BC7FF —— 轻天空蓝，用于插图和辅助氛围
- `accent-peach`: #FFB88C —— 少量暖点缀，用于精选、推荐、温度感信息

#### 功能状态色

- Success：#12B76A
- Warning：#F79009
- Danger：#F04438
- Info：#2E90FA

### 5.3 用色比例建议

- 中性色 / 背景色：约 75%
- 文本和结构色：约 20%
- 品牌与状态强调色：约 5%

### 5.4 渐变使用规则

允许使用轻渐变，但只可用于：

- 首页 Hero 的局部氛围底
- 重点推荐模块的轻背景
- 插图占位与专题区背景
- 后台关键摘要卡的轻强调头部

禁止：

- 整页大面积蓝紫渐变
- 所有卡片统一带渐变
- 所有按钮都使用高亮渐变
- 用渐变替代真实层级设计

---

## 6. 留白与布局系统

### 6.1 核心原则

新版布局的关键不是“更多模块”，而是“更合理的呼吸”。

- 页面外边距要大于上一版
- 首屏和主模块之间要有明显节奏变化
- 卡片内部空间应更松，不做密集堆叠
- 列表页也要保留舒适留白，不做传统后台表格压缩感

### 6.2 容器建议

#### 前台

- 页面最大宽度：1200~1280
- 阅读正文列宽：680~760
- 首页首屏更开放，不要用厚重大容器完全包住
- 首页模块之间建议使用 32 / 40 / 56 的节奏差

#### 后台

- 内容最大宽度：1280~1360
- 侧边导航宽度：224~240
- 工作台首屏更强调横向扫描效率
- 表格型页面也要有充足的顶部摘要与工具区留白

### 6.3 间距体系建议

- 区块间距：24 / 32 / 40 / 56
- 卡片内边距：16 / 20 / 24 / 28
- 表单行距：16 / 20
- 标题与描述之间：8 / 10 / 12

---

## 7. 字体与层级系统

### 7.1 原则

新版不要用“技术感很重的标题 + 大量等宽字体”主导全站。

应转向：

- 标题更清晰、现代、利落
- 正文字体更适合阅读
- 等宽字体只用于少量元信息、slug、时间、统计值

### 7.2 建议字体方向

- **标题字体**：现代中性黑体，强调干净、年轻、克制
- **正文字体**：高可读中文 UI 字体
- **辅助等宽字体**：只用于少量技术元信息，不进入主内容层

### 7.3 层级建议

- Hero 主标题：44~60
- 页面一级标题：30~38
- 模块标题：22~28
- 卡片标题：18~22
- 正文：15~17
- 辅助信息：12~14

### 7.4 字体使用规则

- 标题更短、更利落，避免旧版那种“厚重宣言式”语气
- 正文优先可读性，不追求技术风装饰
- 元信息可适当使用中等字重与更浅颜色，而不是全部大写或高字距

---

## 8. 卡片系统

### 8.1 新版卡片原则

卡片应该是辅助信息组织的工具，而不是页面主角。

新版卡片要做到：

- 更轻
- 更白
- 更少装饰
- 更依赖留白与边线，而不是重阴影和深底

### 8.2 卡片分层建议

1. **主卡片**：白底、轻边框、柔和阴影，用于核心内容
2. **次卡片**：浅底、弱边框，用于辅助信息
3. **功能卡片**：用于后台摘要和统计，强调信息扫描
4. **提示卡片**：用于说明、空状态、提醒

### 8.3 卡片视觉规则

- 圆角建议：16 / 20 / 24
- 阴影要短、浅、柔，不要压重
- hover 更偏背景与边界变化，不要明显悬浮飞起
- 卡片标题、摘要、元信息之间要有稳定间距节奏

---

## 9. 导航系统

### 9.1 前台导航

前台导航应更轻，不要像后台按钮排布。

建议：

- 顶部导航更简洁，减少胶囊按钮感
- 当前态更偏文字高亮或底边强调
- 重要 CTA 保留一个即可，例如“进入后台”或“搜索”
- 首页头部更像内容品牌头图，而不是控制台头部

### 9.2 后台导航

后台左侧导航应体现“清晰工作台”，而不是“深色命令舱”。

建议：

- 侧栏用浅底分区或白底侧栏
- 当前项采用轻色块高亮，不用发光轮廓
- 图标若引入，应统一细线风格
- 顶栏更多承担页面上下文与快速动作，而不是大标题压场

---

## 10. 按钮系统

### 10.1 按钮语义

按钮仍建议分四类：

1. 主按钮
2. 次按钮
3. 危险按钮
4. 文字按钮

### 10.2 新版按钮风格

- 主按钮：品牌蓝实底，清晰但不刺眼
- 次按钮：白底 / 浅底 + 细边框
- 危险按钮：浅红边框或浅红底，不做强烈警告块
- 文字按钮：用于页内轻跳转与次要操作

### 10.3 设计规则

- 高度建议 36~42
- 圆角建议 12~9999
- hover 应以明度变化、边框增强、极轻上浮为主
- 禁止所有按钮都做强发光和强渐变
- 单个视图只保留 1 个明显主按钮焦点

---

## 11. 输入框与表单系统

### 11.1 输入框方向

输入框应更接近现代内容产品与工作台，而不是深色后台表单。

建议：

- 默认白底或极浅灰底
- 细边框清晰可见
- focus 用浅蓝边框 + 柔和外圈
- 标签在上，输入框在下
- 占位符只能辅助提示，不能替代标签

### 11.2 表单布局

- 设置页尽量使用分组表单，而不是长滚动堆叠
- 开关项与文本项分层展示
- 每组设置附带说明、影响范围、风险提示
- 操作反馈靠近表单主操作区

---

## 12. 状态样式

### 12.1 原则

状态色必须稳定，但不能太脏、太重、太像运维后台。

### 12.2 建议

- 成功：浅绿底 + 绿色文字
- 警告：浅橙底 + 橙色文字
- 危险：浅红底 + 红色文字
- 草稿 / 中性：浅灰蓝底 + 中性文字
- 审核中 / 待处理：浅蓝底 + 蓝色文字

### 12.3 使用规则

- 状态标签尺寸一致
- 不同状态只改颜色语义，不改结构
- 列表中状态应快速扫描，不能跳动
- 风险状态可额外配小图标，但不要放大成警报界面

---

## 13. 动效原则

### 13.1 总原则

新版动效必须体现“轻盈”。

不是炫技，不是科技感，不是花哨悬浮，而是：

- 更顺
- 更轻
- 更短
- 更像真实产品

### 13.2 推荐动效

- 模块进入：淡入 + 上移 6~12px
- 卡片 hover：背景微变 + 极轻上浮
- 标签切换：背景和文字色平滑变化
- 面板展开：opacity + translateY
- 表单反馈：边框高亮 + 微弱提示动画

### 13.3 时长建议

- 微交互：120~160ms
- 面板切换：180~240ms
- 页面区块进入：240~320ms

### 13.4 禁止项

- 强发光 hover
- 大面积 blur 浮层动画
- 持续漂浮、粒子、背景移动
- 所有元素都动画
- 过多弹跳、缩放、科技风过场

---

## 14. 前后台共通设计语言

### 14.1 共通点

前后台必须共享：

- 一套基础中性色
- 一套品牌强调色
- 一套字体层级体系
- 一套路由头部 / 页面头部节奏
- 一套按钮、输入框、状态标签规则
- 一套轻卡片和浅边界规则

### 14.2 差异点

#### 前台

- 更强调内容浏览、阅读节奏、专题感
- 更允许大留白、轻氛围底和更柔和的首屏设计
- 更突出标题、摘要、推荐关系

#### 后台

- 更强调信息扫描、批量判断、操作明确
- 摘要区和工具区结构要更强
- 列表和表单更规整，但不能回到老式后台模板感

---

## 15. 关键页面新版改版要点

## 15.1 首页

### 当前问题

- 首页虽然已有重组，但整体仍然带有深色技术编辑部语义
- Hero 和精选卡片偏厚重
- 分类、标签、统计都被强容器包裹，显得压
- 更像“深色内容首页”，还不是“清爽现代内容产品首页”

### 新版改版要点

1. **首页首屏改为更轻的内容品牌头图**
   - 减少厚重边框与深色大容器
   - 用浅底 + 局部柔和渐变氛围
   - 让标题、摘要、搜索、精选入口自然展开

2. **首页主推区从重卡片改为轻专题推荐**
   - 主推文章可以有更大的标题和更开放的布局
   - 不必再套一个厚重右侧深卡
   - 推荐内容更像编辑推荐，而不是控制台模块

3. **最新文章区增强内容流感**
   - 主推 1 篇 + 次级 4 篇仍然成立
   - 但次级卡更轻、更白、更像内容卡片瀑布

4. **分类与标签区弱化“组件盒子感”**
   - 分类更像专题入口
   - 标签更像热词网络或话题导航
   - 避免每个块都一层重背景

5. **首页增加“站点价值感”而不是“统计感”**
   - 公开文章数量、分类数量可保留，但应弱化为辅助信息
   - 首屏主角应是内容主张和精选阅读

---

## 15.2 文章详情页

### 当前问题

- 文章头图仍偏深色、偏厚
- 主体与侧栏关系较稳，但视觉呼吸不够轻
- 评论区已有结构，但仍更像后台模块拼装

### 新版改版要点

1. **文章头部改为轻编辑式头区**
   - 标题、摘要、分类、作者、发布时间、阅读时长更自然排布
   - 背景改为浅色专题头，不再使用重深色渐变块

2. **正文区做成真正的阅读画布**
   - 白底正文卡或接近纸张感的浅底区
   - 控制行宽，提升段落呼吸
   - markdown 样式应更清朗、少厚重装饰

3. **侧栏辅助区弱化“卡片堆叠感”**
   - 阅读信息、目录、继续浏览合并为轻侧栏模块
   - 不必每块都做明显边框盒子

4. **评论区改为真正的讨论模块**
   - 评论项更像对话卡片而不是管理面板
   - 回复层级更轻，避免边框过多
   - 表单应更接近内容社区而不是后台表单

5. **增加内容延展感**
   - 相关文章、上下篇、同类主题入口应更自然嵌入页尾

---

## 15.3 后台仪表盘

### 当前问题

- 统计卡虽然清晰，但依旧较深、较厚、较像传统后台首页
- 待处理事项和最近列表仍在深色卡内嵌套
- 信息有结构，但气质偏旧式 dashboard

### 新版改版要点

1. **仪表盘整体转为浅色工作台**
   - 使用浅背景、白卡、轻边线
   - 降低仪表盘压迫感

2. **统计卡改为轻摘要卡**
   - 数字仍突出，但不做重色块并排压屏
   - 增强标题、趋势、说明的轻量层次

3. **待处理事项区提升优先级**
   - 用更清晰但不刺眼的浅色提示卡承接
   - 帮用户 3 秒内知道“先做什么”

4. **最近文章 / 最近评论改成工作清单**
   - 更像任务流，而不是展示列表
   - 操作按钮收敛、层级更轻

5. **增加“今日工作感”**
   - 例如今日待办、内容状态、审核提醒的简化视图

---

## 15.4 后台文章管理页

### 当前问题

- 功能完整，但表单条、摘要卡、文章项都偏重
- 标题是主焦点这件事是对的，但整体外壳仍旧偏深后台

### 新版改版要点

1. **页面头部更像内容工作页**
   - 标题、描述、主操作清楚
   - 摘要数据更轻量，不抢列表主位

2. **筛选条转为轻工具条**
   - 搜索框、状态筛选、快捷操作一字排开
   - 像工作流工具区，不像表单区块

3. **列表项更像文章卡片行**
   - 标题、slug、摘要、分类、状态、更新时间形成稳定阅读顺序
   - 让编辑者能快速“扫内容”，不是只扫字段

4. **弱化重边框与深底叠层**
   - 通过留白和信息对齐提升清晰度
   - 不靠多层容器做层级

5. **为未来批量操作预留轻入口**
   - 可放在顶部选择态工具条，而不是默认塞进每一项尾部

## 15.5 后台评论审核页

### 当前问题

- 审核页结构已经接近工作台，但视觉仍旧偏深色审核面板
- 状态按钮、详情区、列表区都有些厚
- 风险感表达过于靠色块，不够精致

### 新版改版要点

1. **审核池采用轻量双栏工作台**
   - 左侧为评论流
   - 右侧为详情与审核动作
   - 整体像审阅界面，而不是风控大屏

2. **顶部状态概览更收敛**
   - 用轻摘要卡显示总量、待审核、通过、垃圾评论
   - 不做大面积深色统计区

3. **评论项强调可扫描性**
   - 评论人、内容摘要、所属文章、时间、状态要一眼读懂
   - 当前选中项用轻背景高亮

4. **详情区模块化但不厚重**
   - 原文、父评论、来源信息、审核备注分块展示
   - 动作区固定清晰，颜色语义稳定

5. **风险提示改为温和的语义块**
   - 用浅橙 / 浅红提示即可
   - 不做强烈警报风格

## 15.6 后台设置页

### 当前问题

- 当前设置页仍以大表单块堆叠为主
- 分组虽然存在，但视觉结构还偏传统后台表单
- 右侧概览更像调试信息而不是产品化配置摘要

### 新版改版要点

1. **设置页改为配置中心视角**
   - 左侧分组导航 / 顶部分段切换
   - 中间主表单
   - 右侧说明与摘要

2. **每个分组都要形成“设置卡片组”**
   - 标题、说明、影响范围、输入项分层
   - 开关项与文本项拆开排

3. **保存机制更产品化**
   - 保留固定保存条或页面底部保存区
   - 明确“未保存变更”状态

4. **右侧不再只是接口回显**
   - 概览应变成“当前站点配置摘要”
   - 联调信息可作为二级说明块保留，但不要抢主界面

5. **表单视觉全面轻化**
   - 统一白底输入框、浅灰区块、柔和边线和更大的上下节奏

## 16. 旧风格移除清单

以下风格应在后续实现中逐步移除：

### 16.1 色彩与背景

- 全站默认深色背景
- 大面积黑蓝底 + 青蓝高光

- 所有模块统一大圆角深卡片
- 明显发光边框、hover 发光、强内阴影
- 过于强调 editor-kicker 的科技感小标题

### 16.3 页面语义

- “技术编辑部 / 深夜控制台 / 冷静但不冷淡”作为主视觉叙事
- 后台首页像大屏 dashboard
- 前台首页像控制台内容页

### 16.4 文案调性

- 过于重、过于“宣言式”的标题句法
- 过多“控制台、工作台、模块、面板”这类后台语义占据前台

## 18. 实施原则

### 18.1 先改设计基础，再改页面

优先顺序：

1. 全局色彩 token 轻化
2. 布局壳层轻化
3. 卡片 / 按钮 / 输入框 / 状态统一轻化
4. 首页与文章详情先落地
5. 后台仪表盘与文章管理再跟进

### 18.2 不大规模改代码结构

本轮先完成文档重定义，不做大面积代码重写。

### 18.3 所有后续实现必须遵守两条底线

1. **避免回到沉闷厚重深色风**
2. **避免做成普通白色后台模板**

新版要的是：

- 明亮
- 清爽
- 年轻
- 有内容产品感
- 有秩序但不古板

---

## 19. 结论

新版 UI 的本质不是“把深色改成浅色”这么简单。

真正要完成的是一次设计立场切换：

- 从深色技术控制台，切到明亮现代内容产品
- 从厚重容器感，切到轻盈内容感
- 从后台模板气质，切到年轻清爽的产品气质

最终我们希望前台像一个愿意停留阅读的内容站点，后台像一个愿意长期使用的轻运营工作台。

这才是本次清爽版 UI 规范的目标。
