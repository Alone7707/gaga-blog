# 模块拆分与协作规范

## 1. 开发原则

- 一个模块一提交
- 文档先行，接口先对齐，再开始编码
- 前后端边界先固定，再分别推进
- 任何敏感凭据不入库

## 2. 建议模块拆分

### 前端模块

1. 项目初始化与基础布局
2. 前台首页与文章列表
3. 文章详情页与 Markdown 渲染
4. 分类 / 标签 / 搜索页
5. 后台登录页
6. 后台文章管理
7. 后台分类标签管理
8. 后台评论审核
9. 后台站点设置

### 后端模块

1. 项目初始化与基础框架
2. SQLite + Prisma 数据层
3. Auth 鉴权模块
4. 文章模块
5. 分类模块
6. 标签模块
7. 评论模块
8. 站点设置模块
9. 搜索模块
10. Swagger 与基础测试

## 3. Git 规范

### 提交粒度

每完成一个独立模块，至少执行一次 commit。

### commit message 建议格式

- `docs: add blog product prd`
- `feat(frontend): init app shell`
- `feat(frontend): complete post detail module`
- `feat(backend): add auth module`
- `feat(backend): add post module`
- `refactor(frontend): optimize router guards`

## 4. 协作流程

1. 先写模块需求
2. 再补接口约定
3. 再开始编码
4. 完成模块后立即 commit
5. 模块通过自测后进入下一个模块

## 5. 当前执行要求

- 产品文档已先行落地到 `docs/`
- 前端 agent 负责输出前端模块需求与页面边界
- 后端 agent 负责输出后端模块需求与接口边界
- 后续实现阶段，严格执行“每模块一提交”

## 6. 远程推送说明

当前若未配置远程仓库 URL，则先进行本地 commit。
待补充仓库 URL 后，再统一 push。
