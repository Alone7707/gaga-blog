# UI 改版任务单（清爽版）

## 1. 文档目的

本任务单用于承接《UI 设计规范（清爽版）》，将新的视觉方向拆解成可执行的改版任务。

这一次的重点不是继续优化上一轮深色方案，而是**明确切换设计方向**：

- 从厚重深色改为明亮清爽
- 从后台感改为现代内容产品感
- 从容器主导改为内容主导
- 从科技风装饰改为轻层级与高可读性

原则：

- 先改基础，再改页面
- 不大规模推翻前端结构
- 优先改最影响产品气质的页面
- 让前后台都统一切换到清爽年轻版语言

---

## 2. 当前现状判断

基于现有文档与代码，当前项目已经形成以下现实：

1. 已有一套完整的深色设计 token 和组件壳层
2. 首页、详情页、后台仪表盘、文章管理、评论审核、设置页都已经围绕深色方向实现了一轮样式
3. 当前 UI 已经比默认后台更完整，但审美方向明显偏：
   - 深色
   - 重卡片
   - 冷科技
   - 控制台化
4. 这导致产品虽然“整齐”，但还不够：
   - 年轻
   - 清爽
   - 轻盈
   - 现代内容产品感

因此本轮改版的本质不是修修补补，而是：

> **停止沿用旧版气质，在现有结构上重建新版视觉系统。**

---

## 3. 本轮改版目标

### 3.1 总目标

把当前项目从“深色技术编辑部 / 控制台风格”切换为：

**年轻、简约、清爽、轻盈、现代内容产品感。**

### 3.2 具体目标

1. 重建浅色主世界观
2. 重建轻卡片与轻边界组件体系
3. 重建更清晰的标题与正文层级
4. 重建首页与详情页的内容感
5. 重建后台核心页面的轻工作台感
6. 明确移除旧版厚重深色语义

---

## 4. 必须删除或替换的旧风格

以下旧风格在后续实现时应优先处理：

### 4.1 全局层

### 4.3 页面层

- 首页的大面积深色 Hero
- 详情页深色头图区和侧栏卡片堆叠
- 仪表盘深色统计卡矩阵
- 文章管理深色摘要卡和深色列表项

## 6. 基础层任务

## 6.1 任务 A1：重建全局 Token

### 目标

把当前深色 token 切换为清爽版浅色 token，为后续所有页面提供统一基础。

### 建议改动

- 重构 `frontend/src/style.css` 中的全局颜色变量
- 将主背景从深色改为浅色体系
- 新增轻背景、白卡片、柔和边线、浅状态色变量
- 将阴影改为更短、更浅、更柔和
- 降低统一圆角厚重感

### 涉及文件

- `frontend/src/style.css`
- 如有必要可新增：
  - `frontend/src/styles/tokens.css`
  - `frontend/src/styles/theme-light-content.css`

### 验收标准

- 全站默认背景不再是深色
- 组件不再依赖深色渐变背景变量
- token 能同时支持前台和后台清爽版样式

---

## 6.2 任务 A2：重做全局基础类

### 目标

让公共样式从“深色面板系统”转为“明亮轻层级系统”。

### 建议改动

重点改以下基础类：

- `.app-shell`
- `.panel-surface`
- `.panel-subtle`
- `.editor-kicker`
- `.editor-title`
- `.ui-btn*`
- `.ui-input / .ui-select / .ui-textarea`
- `.ui-badge*`
- `.article-prose`

### 核心方向

- 去掉深色玻璃感
- 降低大面积渐变和阴影
- 改成白底 / 浅底 / 细边框 / 柔和阴影
- 将科技感弱化为现代内容产品感

### 验收标准

- 不看页面，只看基础类，也能感知视觉方向已切换
- 所有组件不再默认呈现深色控制台语义

---

## 6.3 任务 A3：重做 SectionCard

### 目标

把当前厚重统一卡片改为更轻的通用容器。

### 建议改动

- 保留变体机制，但重新定义视觉：
  - hero：轻专题卡
  - panel：白卡片 / 浅卡片
  - dashed：浅提示卡
  - default：通用内容卡
- header 区域边界更轻
- 降低阴影和背景存在感
- 更依赖间距而不是强壳层

### 涉及文件

- `frontend/src/components/common/SectionCard.vue`

### 验收标准

- 卡片能明显看出主次，但不再厚重
- 单页多个卡片并排时不显得闷、不压

---

## 6.4 任务 A4：重做按钮、输入框、状态标签

### 目标

把现有公共交互控件统一切换到清爽版风格。

### 具体任务

#### 按钮

- 主按钮改为清晰品牌蓝，不使用强发光渐变
- 次按钮改为白底或浅灰底边框按钮
- ghost 按钮改为轻品牌底而不是科技按钮

#### 输入框

- 改为白底 / 浅底输入框
- focus 用浅蓝边框和柔光外圈
- 去掉深底表单感

#### 状态标签

- 改为浅色底 + 语义色文字
- 统一尺寸与间距
- 避免像后台警报标签

### 验收标准

- 页面中所有基础交互控件风格统一
- 轻量、现代、清爽，不再厚重

## 7. 前台页面任务

## 7.1 首页改版

### 优先级
P0

### 改版目标

把首页从“深色技术编辑部首页”切换为“现代内容产品首页”。

### 具体任务

#### H1：重做首页 Hero

- 弱化深色大容器
- 改为更开放的浅色首屏
- 标题更简洁，减少宣言式厚重语气
- 搜索、精选入口、站点定位更自然组合

#### H2：重做精选主推区

- 取消当前偏厚重的右侧深色精选主卡
- 改为轻专题推荐布局
- 主推文章可以更像编辑推荐而不是“重点模块”

#### H3：轻化统计信息

- 文章数、分类数、标签数可保留
- 但改成辅助信息，不再和 Hero 等权竞争

#### H4：重做最新文章区

- 主文章 + 次级文章节奏保留
- 改成白卡 / 浅卡组合
- 标题、摘要、时间、标签的轻层级更明确

#### H5：重做分类与标签区

- 分类更像内容专题入口
- 标签更像话题索引，而不是统一胶囊矩阵
- 降低盒子感和边框感

- `frontend/src/pages/public/HomePage.vue`
- `frontend/src/components/public/PostCard.vue`
- `frontend/src/layouts/PublicLayout.vue`

### 验收标准

- 首页一眼看上去是现代内容产品，而不是深色博客控制台
- 首屏明显更轻、更年轻、更有呼吸感
- 内容成为主角，容器退居辅助

## 13. 页面优先重做顺序

如果实现资源有限，建议严格按以下顺序推进：

1. **首页**
2. **文章详情页**
3. **后台仪表盘**
4. **后台文章管理页**
5. **后台评论审核页**
6. **后台设置页**

原因：

- 首页决定产品第一印象
- 详情页决定阅读体验

## 14. 结论

本次清爽版改版，不是简单换配色，而是一次明确的产品气质切换。
