# 技术选型与架构

## 1. 技术选型

### 前端

- Vue 3
- Vite
- Vue Router
- Pinia
- UnoCSS 或 Tailwind CSS（二选一，优先 UnoCSS，轻量且适合快速产出）
- Markdown 渲染：markdown-it

### 后端

- Node.js
- NestJS
- Prisma ORM
- SQLite
- JWT 鉴权
- Swagger（接口文档）

> 选择 NestJS 的原因：模块化清晰，适合做中长期维护；如果实现阶段追求极简，也可切换为 Fastify + 手动分层。

## 2. 推荐项目结构

```text
/blog-app
  /docs
  /frontend
  /backend
```

### frontend

```text
frontend/
  src/
    api/
    assets/
    components/
    layouts/
    pages/
    router/
    stores/
    utils/
```

### backend

```text
backend/
  src/
    modules/
      auth/
      post/
      category/
      tag/
      comment/
      setting/
      search/
    common/
    prisma/
```

## 3. 核心后端模块

- auth：管理员登录、鉴权、中间件
- post：文章 CRUD、发布、slug、摘要
- category：分类管理
- tag：标签管理
- comment：评论提交、审核、查询
- setting：站点配置
- search：搜索能力

## 4. 核心前端模块

- public-web：前台站点
- admin-web：后台管理
- shared-ui：复用组件层
- content-render：Markdown 渲染与目录

## 6. API 设计原则

- RESTful 为主
- 后台接口统一前缀 `/api/admin`
- 前台接口统一前缀 `/api/public`
- 返回结构统一
- 分页、排序、筛选参数统一规范

## 7. 安全与约束

- 管理员接口必须鉴权
- 评论提交接口需要限流
- 敏感配置不入库明文展示
- PAT、密钥、密码禁止写入代码仓库

## 8. 可扩展方向

- 数据库从 SQLite 平滑升级到 PostgreSQL
- 主题机制
- 插件机制
- OpenAPI SDK 自动生成
