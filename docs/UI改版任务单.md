# UI 改版任务单

## 1. 文档目的

本任务单用于把《UI 设计规范》落成可执行的前端改版计划，适合作为后续实现排期、拆分 issue、提交 commit 和联调验收的依据。

原则：

- 优先改“骨架页面”和“公共样式能力”
- 不做无边界翻新
- 先统一视觉语言，再逐页落地
- 每个页面改版都要有明确目标、改动项、组件依赖、验收标准

---

## 2. 当前前端现状判断

基于现有代码结构，项目已具备以下基础：

- public / admin 双布局已完成
- 首页、详情页、搜索页、后台仪表盘、文章管理、评论审核、设置页均已有页面骨架
- UnoCSS 已接入，可支撑快速重构视觉样式
- `SectionCard` 等公共容器已存在，可作为第一批重构对象

当前短板：

1. 缺少全局设计 token
2. 页面主要依赖局部 class 直接堆叠
3. 前台页面更像“功能拼装”，缺少品牌视觉锚点
4. 后台页面信息可用，但还没有内容控制台的识别度
5. 公共组件风格没有形成统一规范

---

## 4. 基础层改版任务

### 任务 A1：建立设计 Token

#### 目标

把当前零散颜色、边距、圆角、阴影抽离成统一变量，为后续页面改版提供基础。

#### 建议改动

- 新建全局 token 文件或在 `src/style.css` 中建立 CSS Variables
- 至少沉淀以下变量：
  - 背景层级
  - 文本层级
  - 强调色
  - 状态色
  - 圆角
  - 间距
  - 阴影
  - 动效时长

#### 涉及文件

- `frontend/src/style.css`
- 可新增：`frontend/src/styles/tokens.css`
- 可新增：`frontend/src/styles/theme.css`

#### 验收标准

- 页面不再直接依赖大量硬编码十六进制颜色
- 前后台共用同一套基础 token
- 可支持后续组件统一替换样式

---

### 任务 A2：升级基础容器组件

#### 目标

把 `SectionCard` 从普通卡片升级为具备统一层级能力的通用容器。

#### 建议改动

- 支持变体：默认 / 强强调 / 虚线说明 / 数据面板
- 支持头部区域样式分级
- 支持 action 槽位统一对齐
- 支持内边距尺寸变体

#### 涉及文件

- `frontend/src/components/common/SectionCard.vue`
- 可新增：标题条、状态徽标、面板头组件

#### 验收标准

- 公共卡片不再只是一种样式
- 页面改版时可明显减少重复样式

---

### 任务 A3：统一按钮、输入框、状态标签样式

#### 目标

把当前分散在页面中的按钮和输入框样式收敛成统一规则。

#### 建议改动

- 建立按钮语义：主按钮 / 次按钮 / 危险按钮 / 轻按钮
- 建立输入框 focus 规则
- 建立状态标签颜色映射规则
- 后台状态：草稿 / 已发布 / 已归档 / 待审核 / 已通过 / 驳回 / 垃圾评论统一

#### 验收标准

- 同类操作外观一致
- 后台状态颜色不再在多个页面分别维护

---

## 5. 前台页面改版任务

## 5.1 首页改版

### 优先级
P0

### 当前问题

- 缺少真正的首页首屏
- 分类、标签、最新文章都像普通列表块
- 没有建立站点内容主张

### 改版目标

把首页改成“开源博客产品首页”，而不是“文章接口演示页”。

### 具体任务

#### H1：重做首页 Hero 区

- 增加站点定位语句
- 增加主标题、副标题
- 增加搜索入口和核心 CTA
- 右侧加入精选文章或主视觉内容卡

#### H2：重组首页内容结构

建议结构：

1. Hero
2. 精选 / 最新主文章
3. 最新文章列表
4. 分类导航
5. 热门标签 / 搜索入口
6. 站点理念 / 开源能力说明

#### H3：升级文章卡片样式

- 区分主推卡与常规卡
- 增加元信息排版层级
- 优化标题、摘要、标签、日期关系

#### H4：增强标签区层级

- 不再是统一小胶囊平铺
- 根据热度或文章数做层级
- 支持更强的 hover 反馈

### 涉及文件

- `frontend/src/pages/public/HomePage.vue`
- `frontend/src/components/public/PostCard.vue`
- `frontend/src/layouts/PublicLayout.vue`

### 验收标准

- 首页首屏一眼可识别为博客产品首页
- 至少存在 1 个强视觉模块和 2 个明显信息层级变化
- 最新文章与分类/标签不再同权平铺

---

## 5.2 文章详情页改版

### 优先级
P0

### 当前问题

- 文章头部不够强
- 正文阅读区和评论区关系平均
- 缺少阅读辅助和继续浏览机制

### 改版目标

做成真正适合技术长文阅读的详情页。

### 具体任务

#### D1：重做文章头部

- 标题、摘要、分类、标签、时间、作者形成完整头部
- 引入更强的视觉节奏与留白
- 若有封面图，建立封面占位规范

#### D2：重做正文布局

- 正文列宽收敛
- 优化 markdown 内容样式：标题、段落、引用、代码块、列表、表格
- 增加段落间距与可读性

#### D3：增加右侧阅读辅助区

- 目录导航
- 阅读信息
- 标签快速跳转
- 相关文章 / 继续浏览入口

#### D4：重做评论区

- 评论列表改为更像讨论区
- 表单样式统一
- 回复逻辑与父评论关系更清晰

#### D5：补充上下文导航

- 上一篇 / 下一篇
- 返回首页 / 分类页入口

### 涉及文件

- `frontend/src/pages/public/PostDetailPage.vue`
- 可新增：`TocNav`、`ArticleMeta`、`CommentCard`

### 验收标准

- 正文阅读区域明确成为视觉核心
- 辅助区不抢主内容，但足够有用
- 评论区从“表单+列表”升级为“讨论模块”

## 5.3 分类页 / 标签页 / 搜索页改版

### 优先级
P1

### 当前问题

- 页面语义差异弱
- 都像同一页面换标题

### 改版目标

让三类页面共享骨架，但拥有不同语义头部。

### 具体任务

#### C1：分类页改版

- 头部强调当前分类定位与文章数量

- 标签可做大小、透明度或描边差异
- 当前标签结果页强调话题关联

#### C3：搜索页改版

- 搜索框升级为首要交互中心
- 显示关键词、命中数、搜索说明
- 高亮关键词命中信息
- 空结果态给出下一步建议

### 涉及文件

- `frontend/src/pages/public/CategoriesPage.vue`
- `frontend/src/pages/public/CategoryPostsPage.vue`
- `frontend/src/pages/public/TagsPage.vue`
- `frontend/src/pages/public/TagPostsPage.vue`
- `frontend/src/pages/public/SearchPage.vue`

### 验收标准

- 三类页面具备统一结构，但不再视觉同质化
- 搜索页有明确“检索工作台”感

### 优先级
P0

### 当前问题

- 统计卡、最近文章、最近评论都在，但不够像真正首页
- 缺少待办导向

### 改版目标

让后台首页先告诉管理员：现在站点状态如何、你最应该做什么。

### 具体任务

#### A-D1：重构首屏模块

建议首屏结构：

1. 运营概览卡组
2. 待处理事项
3. 快捷操作入口

#### A-D2：升级统计卡

- 数字更有主次
- 增加状态说明和趋势感提示
- 卡片高光不一样，避免一排同样卡片

#### A-D3：重组最近文章 / 最近评论

- 更像工作列表
- 显示下一步操作
- 强化待审核评论优先级

#### A-D4：增加站点状态模块

- 可放轻量系统状态或内容节奏信息
- 如“最近更新”“待处理评论”“站点设置完成度”

### 涉及文件

- `frontend/src/pages/admin/AdminDashboardPage.vue`

### 验收标准

- 首屏可以在 5 秒内读懂后台当前状态
- 至少有一个“待处理”模块优先于普通列表展示

## 6.3 后台文章管理页改版

### 优先级
P0

### 当前问题

- 列表有用，但操作场景不够聚焦
- 表格信息密度高但缺视觉锚点

### 改版目标

做成后台最强主工作页。

### 具体任务

#### A-P1：增加页面摘要区

- 显示总文章数、草稿数、已发布数、最近更新时间

#### A-P2：升级筛选条

- 搜索、状态筛选、分类筛选、快捷新建形成统一工具条
- 更像工作台，不像表单拼装

#### A-P3：升级表格/列表结构

- 标题信息作为主焦点
- slug、摘要、标签、状态、时间形成稳定层级
- 强调“编辑”和“预览”操作

#### A-P4：加入批量操作预留

即便后端尚未接，也可以先预留 UI 位。

#### A-P5：响应式降级

- 在窄屏下退化为文章卡片列表

### 涉及文件

- `frontend/src/pages/admin/AdminPostsPage.vue`

### 验收标准

- 文章管理页可一眼扫描出重点文章和状态
- 筛选条、摘要区、列表主体三层关系清楚

## 10. 页面优先级结论

如果前端实现资源有限，建议按以下顺序推进：

### 第一优先级

1. 首页
2. 文章详情页
3. 后台仪表盘
4. 后台文章管理页

原因：

- 这四页最能决定产品第一印象
- 同时覆盖读者主链路和管理员主链路
- 能最快建立产品调性和设计系统骨架

### 第二优先级

5. 搜索页
6. 评论审核页
7. 设置页

### 第三优先级

8. 分类页
9. 标签页
10. 归档页等补充页面

---

## 11. 结论

本次改版不建议直接进入“大面积手调样式”，而应先建立统一基础，再按主链路页面落地。最关键的是先把：

- 首页做出站点气质
- 文章详情做出阅读体验
- 仪表盘做出工作台感
- 文章管理做成后台主战场

这四页一旦立住，整套开源博客产品的 UI 方向就会稳定下来，后续页面都可以沿着同一语言快速扩展。
