# 开源博客产品 PRD

## 1. 产品定位

一个可自托管、可二次开发、默认对个人开发者友好的开源博客系统。

目标不是做“大而全”的 CMS，而是做“开箱可用、结构清晰、易于扩展”的博客产品。

## 2. 目标用户

### 2.1 核心用户

- 独立开发者
- 技术博主
- 小型内容团队
- 想快速搭建个人站的开发者

### 2.2 典型诉求

- 快速搭建博客
- 稳定写作与发布
- 文章检索与归档
- 具备基础 SEO
- 可自定义主题
- 低成本部署与维护

## 3. 产品目标

### 3.1 第一阶段目标（MVP）

- 支持管理员登录
- 支持文章创建、编辑、删除、发布
- 支持分类、标签、草稿
- 支持公开文章列表与详情页
- 支持评论提交与后台审核
- 支持站点基础设置
- 支持搜索与 SEO 基础能力

### 3.2 非目标

- 不在第一阶段支持复杂工作流审批
- 不在第一阶段支持多租户
- 不在第一阶段支持强社交关系链
- 不在第一阶段支持所见即所得复杂排版编辑器

## 4. 核心价值

- 对开发者友好：部署简单、结构直观、扩展成本低
- 对作者友好：内容管理顺手，写作链路短
- 对读者友好：访问快、搜索快、阅读体验好

## 5. 核心功能模块

### 5.1 博客前台

- 首页文章流
- 文章详情页
- 分类页
- 标签页
- 搜索页
- 关于页 / 自定义静态页
- 归档页

### 5.2 后台管理

- 登录 / 登出
- 仪表盘
- 文章管理
- 分类管理
- 标签管理
- 评论审核
- 站点设置

### 5.3 内容系统

- Markdown 内容存储
- 草稿 / 已发布状态
- 摘要
- 封面图
- SEO 字段（title、description、slug）
- 发布时间

### 5.4 评论系统

- 游客评论
- 敏感词 / 垃圾评论基础拦截
- 评论审核状态
- 回复能力（一期可限制为单层回复）

### 5.5 系统能力

- 鉴权
- 审计日志（基础）
- 搜索索引
- 主题配置预留
- RSS / Sitemap

## 6. MVP 功能清单

### P0

- 文章 CRUD
- 分类 / 标签管理
- 前台文章列表 / 详情
- 评论提交 + 审核
- 站点设置
- 搜索
- SEO 基础字段

### P1

- 归档页
- RSS
- Sitemap
- 相关文章推荐
- 访问统计基础版

### P2

- 多主题切换
- 附件上传
- 文章版本记录
- 草稿自动保存

## 7. 用户角色

### 管理员

- 管理站点配置
- 发布与维护内容
- 审核评论
- 管理分类标签

### 游客

- 浏览文章
- 搜索文章
- 提交评论

## 8. 核心流程

### 8.1 发文流程

登录后台 → 新建文章 → 编辑 Markdown → 设置分类标签/SEO → 保存草稿或发布 → 前台可见

### 8.2 评论审核流程

游客提交评论 → 状态为待审核 → 管理员审核通过 / 驳回 → 前台展示

### 8.3 搜索流程

用户输入关键词 → 返回文章标题、摘要、标签匹配结果 → 跳转详情页

## 10. 成功指标

### 产品指标

- 新站点从部署到发布第一篇文章时间 < 30 分钟
- 管理员发文主链路操作步骤 <= 8 步
- 首屏页面响应清晰，基础访问流畅

### 工程指标

- 模块边界清晰
- SQLite 单机部署可稳定运行
- 前后端可独立开发与替换
- 具备基础开源文档

## 11. 风险与对策

### 风险

- SQLite 在高并发场景有上限
- 评论系统易受垃圾内容攻击
- 主题系统若过早设计会拉高复杂度

### 对策

- MVP 先定位单站点、低到中等流量
- 评论加入审核与基础限流
- 主题系统先做配置化预留，不先做完整插件化
