# 开源博客产品文档

本目录用于沉淀该开源博客产品的产品、技术、协作、联调与模块规划文档。

## 文档索引

- `PRD-开源博客产品.md`：产品需求文档
- `技术选型与架构.md`：技术方案、架构边界、核心数据模型
- `模块拆分与协作规范.md`：模块拆分、开发顺序、Git 提交规范
- `开发进度看板.md`：当前项目阶段、已完成范围、剩余重点尾项
- `联调缺口清单.md`：前后端联调结论、当前缺口与后续建议
- `API返回约定与当前状态.md`：接口顶层 envelope、分页字段、公开接口与后台核心接口当前返回结构

## 当前产品目标

打造一个面向个人开发者、小团队与内容创作者的开源博客系统，具备：

- 内容发布与管理
- 标签与分类
- 评论与审核
- 搜索与 SEO
- 后台管理
- 可扩展主题能力
- 轻量部署与低维护成本

## 当前技术方向

- 前端：Vue 3 + Vite + Vue Router + Pinia
- 后端：Node.js + NestJS（如实现阶段评估过重，可降级为 Fastify）
- 数据库：SQLite + Prisma
- 鉴权：JWT + HttpOnly Cookie

## 当前接口文档状态

当前仓库接口文档已完成一轮收口，现状如下：

- 后端成功 / 失败顶层响应已统一为 envelope
- 前端 `frontend/src/api/http.ts` 已统一解包顶层 `data`
- 公开接口主链路已覆盖：
  - 文章列表 / 详情
  - 分类 / 标签列表与对应文章列表
  - 搜索
  - 归档
  - 评论列表 / 提交评论
  - 站点设置 / 站点概览 / About 静态页
- 后台核心接口已覆盖：
  - 仪表盘概览
  - 文章列表 / 详情 / 创建 / 更新 / publish / unpublish / archive
  - 评论统计 / 列表 / 详情 / 审核动作
  - 站点设置
  - 分类 / 标签列表与新增更新

重点说明：

- 顶层 envelope 已统一，但 `data` 内部仍存在“直接对象返回”和“带业务键包裹返回”并存的情况
- 新页面或新联调如涉及详情、动作、统计类接口，应优先核对 `API返回约定与当前状态.md`

## 协作约束

- 每完成一个独立模块，必须提交一次 Git commit
- 文档与代码都应先落仓库，再进入下一个模块
- 不在文档、代码、git config 中持久化 PAT 等敏感凭据
